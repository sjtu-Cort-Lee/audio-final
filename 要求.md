1. 任务描述
给定一段语音信号，系统需要输出对应的文本转写。该方向要求使用语音自监督模型提取语音表征，并构建 ASR 系统。系统可以使用连续 hidden states，也可以进一步构建离散语音单元。具体建模方式不限，可以采用 CTC、encoder-decoder、Transducer、Transformer、Conformer、RNN、语言模型融合或其他结构。
2. 数据要求
推荐使用公开英文 ASR 数据集，例如 LibriSpeech。可根据方法设计自行决定数据预处理方式，包括采样率统一、文本规范化、子词建模、数据增强、特征缓存等。
3. 方法要求
ASR 方向的系统应围绕以下问题展开：
1. 如何有效利用预训练语音自监督表征完成低资源语音识别。
2. 连续表征与离散语音单元对识别性能、模型复杂度和信息压缩效率有何影响。
3. 不同自监督模型、不同表征层或不同单元设计是否会带来明显差异。
4. 在有限标注数据条件下，系统如何取得更好的识别性能与泛化能力。
系统实现具有充分自由度，但应至少包含一个清晰的语音自监督表征使用方案，并与合理的对比系统或消融设置进行比较。
4. 评价指标
ASR 方向以识别准确率为核心评价目标，主要指标包括：
指标
含义
WER
Word Error Rate，词错误率
CER
Character Error Rate，字符错误率
可根据系统设计进一步报告：
指标
含义
推理速度
RTF 或平均推理耗时
token rate
使用离散单元时，平均每秒 token 数
bitrate
使用离散单元时，离散表示的信息率
codebook size
使用离散单元时，离散单元词表大小
5. 建议分析方向
报告中可围绕以下方向展开分析：
1. 不同自监督模型的比较。
2. 不同 hidden layer 的比较。
3. 连续表征与离散单元的比较。
4. 不同 codebook size 的影响。
5. token 去重、时长建模或 BPE 等策略的影响。
6. 低资源条件下训练数据规模对性能的影响。
7. 错误案例分析，例如替换、删除、插入错误。
8. 识别性能与模型大小、训练成本、bitrate 之间的关系。
